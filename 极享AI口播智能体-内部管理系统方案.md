# 极享AI口播智能体 · 内部管理系统项目方案

> 版本：v1.0 | 日期：2026-06-19 | 状态：立项方案
> 基于《极享AI口播智能体作战方案》推导的系统落地规划

---

## 一、项目背景与业务目标

### 1.1 业务链路

```
直播间店播获客 → 899元入门锁客 → 微信销售铁军诊断筛选 → 29800/59800高客单升单 → 交付团队服务复制
```

### 1.2 核心经营指标

| 指标 | 定义 | 系统支撑点 |
|------|------|-----------|
| 店播成交人数 | 直播间持续卖899 | 订单管理、线索捕获 |
| 加微率 | 899用户导入微信私域 | 客户报备、线索流转 |
| 有效诊断率 | 识别自用/B端/服务商/资源型 | 客户标签、画像管理 |
| 高客单升单率 | 推动29800/59800成交 | 销售漏斗、话术SOP |
| 交付案例率 | 成交后出案例+复购 | 案例管理、任务追踪 |

### 1.3 系统设计原则

1. **收入链路驱动** — 每个模块服务成交链路，不是资料库
2. **先跑通再完整** — 分阶段交付，最小闭环先行
3. **数据自动化** — 表单管理底座，总部看清市场真实情况
4. **角色权限分明** — 总部运营/销售铁军/服务商代理/管理层查看

## 六、核心数据模型

### 6.1 关键实体关系

```
Agent (代理)
  ├── id, name, level, city, source, status, performance
  ├── has_many -> Customer (客户)
  ├── has_many -> Order (订单)
  └── has_many -> Commission (佣金)

Customer (客户线索)
  ├── id, name, type, budget, stage, next_action
  ├── belongs_to -> Agent
  ├── has_many -> ReportRecord (报备记录)
  └── has_many -> Order

Order (订单)
  ├── id, product_id, amount, status, refund_status
  ├── belongs_to -> Agent
  ├── belongs_to -> Customer
  └── has_many -> Commission

Commission (佣金)
  ├── id, order_id, agent_id, amount, settlement_status
  └── belongs_to -> Order

Product (产品)
  ├── id, name, price, tier, color_tag, features
  └── has_many -> Order

ContentBlock (内容块 - CMS通用)
  ├── id, module, category, title, content, status
  └── polymorphic 归属

Camp (训练营)
  ├── id, name, type, start_date, end_date
  ├── has_many -> Homework (作业)
  └── has_many -> Trainee (学员)

Case (案例)
  ├── id, type, customer_name, story, results, screenshot
  └── has_many -> CaseMedia (案例素材)
```


## 九、关键技术决策

| 决策项 | 推荐方案 | 理由 |
|--------|---------|------|
| **前端状态管理** | Zustand + React Query | 轻量全局+服务端缓存 |
| **内容管理** | 自建CMS或集成MDX | 灵活配置内容模块 |
| **文件存储** | 阿里云OSS / 腾讯COS | 视频/图片素材分发 |
| **权限模型** | RBAC + 数据行级权限 | 按角色+代理等级控制可见 |
| **表单引擎** | react-hook-form + zod | 类型安全+验证强大 |
| **数据看板** | Recharts + 后端聚合接口 | 实时/定期刷新 |
| **文档协作** | 内置富文本编辑器 | 话术/SOP/政策在线编辑 |

## 十一、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 业务优先级频繁变化 | 开发中断 | 分阶段严格锁定MVP范围 |
