# 首期交付范围与模块状态

更新时间：2026-06-21

本文用于把当前“已做页面”拆成可管理的交付范围。状态说明：

- `可演示`：页面可访问，适合作为产品/业务演示。
- `待接 API`：页面和交互骨架存在，但核心数据仍来自 mock。
- `仅原型`：目前主要用于说明业务意图，不能视为可上线能力。
- `待确认`：需要产品负责人确认首期是否纳入。

## 首期建议边界

建议首期聚焦 6 类能力：

1. 系统管理：用户、角色、菜单权限、操作日志。
2. 数据驾驶舱：经营 KPI、订单/待办摘要。
3. 产品矩阵：产品、权益、交付清单维护。
4. 项目认知：FAQ、竞品、指南内容维护。
5. 销售成交：话术、问答、诊断、报价、SOP。
6. 内容库：案例见证 + 内容素材。

招商代理、获客引流、交付服务、政策合同可以作为首期增强项；培训赋能、品牌资产、数据运营建议放入二期，避免首期战线过长。

## API 优先级

P0：

- 认证登录、当前用户、菜单权限
- 用户、角色、操作日志
- 产品矩阵
- 销售话术、问答、诊断模板、报价方案
- FAQ、竞品、指南
- 案例和素材列表

P1：

- 数据驾驶舱聚合指标
- 招商代理与佣金结算
- 获客动作与渠道数据
- 交付 SOP 与售后记录
- 政策合同模板

P2：

- 培训课程与学习进度
- 品牌资产和证书
- 数据运营复盘、投诉分析、排行榜
